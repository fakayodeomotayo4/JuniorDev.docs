Git Basics

Everything you need to stop being afraid of version control. From zero to confident in one read.

What is Git?

Git is a version control system. Think of it as "save points" for your code. Every time you
make meaningful progress, you save a snapshot. If you break something, you can go back.

NOTE

Git is not the same as GitHub.Git is the tool that tracks changes. GitHub is a website that hosts Git repositories online.

Setting Up

After installing Git, configure your identity. This is attached to every save point you make.

git config --global user.name "Your Name"
git config --global user.email "you@email.com"


# Verify your setup
git config --list

Starting a Repository

A repository (repo) is a folder that Git is tracking. Two ways to start:

# Option 1: Start tracking an existing folder
cd my-project
git init

# Option 2: Clone someone else's repo
git clone https://github.com/user/repo.git

The Core Workflow

Git has three "zones" your files move through:

# Check what's changed
git status

# Stage specific files
git add filename.txt

# Stage everything
git add .

# Save a snapshot (commit)
git commit -m "Describe what you changed"

# View your history
git log --oneline

Pro Tip

Write commit messages in the imperative: "Add login form" not "Added login form." Think of it as completing the sentence: "This commit will..."

Branches

Branches let you work on features without messing up the main code. Think of them as
parallel timelines.

# Create and switch to a new branch
git checkout -b feature-name

# Switch between branches
git checkout main
git checkout feature-name

# See all branches
git branch

# Merge a branch into main
git checkout main
git merge feature-name

# Delete a branch after merging
git branch -d feature-name

Working with Remote (GitHub)

Push your local code to a remote server so others can see it:

# Connect to a remote repository
git remote add origin https://github.com/user/repo.git

# Push your code
git push -u origin main

# Pull changes others have made
git pull origin main

Undoing Mistakes

Everyone messes up. Here's how to fix it:

# Undo changes to a file (before staging)
git checkout -- filename.txt

# Unstage a file (keep changes)
git reset HEAD filename.txt

# Undo the last commit (keep changes)
git reset --soft HEAD~1

# Undo the last commit (discard changes) ⚠️
git reset --hard HEAD~1

Warning

--hard permanently deletes changes. Use with extreme caution. When in doubt, use --soft.
